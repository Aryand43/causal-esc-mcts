The core ideas (from the writeup + supervisor feedback) are:

- Emotional Support Conversations (ESC) are modeled as a finite-horizon MDP  
  \(M = (S, A, P, R)\) where:
  - \(s_t \in S\): dialogue state (history up to turn \(t\))
  - \(a_t \in A\): support strategy
  - \(P(s_{t+1} \mid s_t, a_t)\): emotional state evolution
  - \(R(s_t, a_t)\): affective reward
- A trajectory is \(\tau = (s_0, a_0, ..., s_T)\), with return \(R(\tau) = R(s_T)\),  
  objective: \(\max_\pi \mathbb{E}_{\tau \sim \pi} \sum_{t=0}^T R(s_t, a_t)\).
- We follow **AFlow**: learning from trajectory-level rewards via *flow consistency*.
- We **extend AFlow** with:
  1. **Causal state representation**:
     - A causal graph \(G_t = (U_t, E_t)\) over causes/emotions/effects.
     - Dialogue history \(H_t \in \mathbb{R}^{K \times d_h}\)
     - Causes \(C_t \in \mathbb{R}^{n_c \times d_c}\)
     - Emotion \(e_t \in \mathbb{R}^{d_e}\)
     - Phase \(p_t \in \mathbb{R}^{d_p}\)
     - Full state tensor:
       \[
       s_t = \phi(H_t, G_t) = \text{concat}(\bar{H}_t, \bar{C}_t, e_t, p_t) \in \mathbb{R}^{d}
       \]
  2. **Action representation**:
     - Each action is a strategy–cause pair:
       \[
       a_t = (\sigma, c_i), \quad a_t = \psi(\sigma, c_i) \in \mathbb{R}^{d_a}
       \]
     - Candidate set: \(A(s_t) = \{a_1, ..., a_k\}\) generated by a policy network \(\pi_\theta(a \mid s)\).
  3. **Latency-optimized transition model**:
     - Instead of full LLM rollouts:
       \[
       \hat{s}_{t+1} = f_\theta(s_t, a_t), \quad f_\theta: \mathbb{R}^{d + d_a} \to \mathbb{R}^{d}
       \]
     - Decomposed as:
       \[
       f_\theta(s_t, a_t) = (e_t + \Delta e_t, \text{presolve}, p_{t+1})
       \]
       where:
       - \(\Delta e_t\): predicted emotion change
       - \(\text{presolve} \in [0,1]^{n_c}\): cause resolution probabilities
       - \(p_{t+1}\): next phase embedding
  4. **Reward decomposition**:
     \[
     R(s_t, a_t) = \alpha R_{\text{cause}} + \beta R_{\text{emotion}} + \gamma R_{\text{phase}}
     \]
     with:
     - \(R_{\text{cause}} = \sum_i p^{(i)}_{\text{resolve}}\)
     - \(R_{\text{emotion}} = -\|e_{t+1} - e_t\|\)
     - \(R_{\text{phase}} = \mathbf{1}(p_{t+1} > p_t)\)
  5. **MCTS planning with PUCT**:
     - PUCT selection:
       \[
       U(s, a) = Q(s, a) + c \cdot \pi_\theta(a \mid s) \sqrt{\frac{N(s)}{1 + N(s, a)}}
       \]
     - Rollouts use \(f_\theta\), approximate return:
       \[
       \hat{G}(s_t, a_t) = \sum_{k=1}^{L'} R(\hat{s}_{t+k}, a_{t+k})
       \]
     - Update:
       \[
       Q(s, a) \leftarrow Q(s, a) + \frac{1}{N(s, a)}(\hat{G} - Q(s, a))
       \]
     - This yields \(Q_b(s), Q_b(s, a)\) defining flow:
       \[
       F(s) = Q_b(s) V_\phi(s)
       \]
  6. **Training objective**:
     - Flow-based:
       \[
       L_{\text{flow}} = \sum (F(s \to s') - F(s)\pi_\theta(a \mid s))^2
       \]
     - Ranking:
       \[
       L_{\text{rank}} = \max(0, \gamma - (V_\phi(s, a_w) - V_\phi(s, a_l)))
       \]
     - Joint objective:
       \[
       L = L_{\text{flow}} + \lambda L_{\text{rank}}
       \]
  7. **Inference**:
     - Action:
       \[
       a_t = \arg\max_a \left[\log \pi_\theta(a \mid s_t) + V_\phi(s_t, a)\right]
       \]
     - Final response: a single LLM (Qwen 9B) conditioned on \((s_t, a_t)\).
